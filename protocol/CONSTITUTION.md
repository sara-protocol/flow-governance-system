# SARA Flow Constitution

This constitution defines how the SARA protocol evolves
without losing its governing function.

---

## Constitutional Status

The following principles are constitutional:

1. Intent First
2. Structure Visible
3. Right to Exit
4. Recursive Accountability

They may not be removed.

---

## Versioning Rules

### MAJOR (v1 → v2)
- Changes the nature of cognitive governance
- Requires founder approval
- Requires public rationale

### MINOR (v1.0 → v1.1)
- Clarifies enforcement
- Adds metrics
- Tightens constraints
- MUST NOT weaken protections

### PATCH (v1.0.1)
- Language clarification
- Implementation notes
- No behavioural impact

---

## Prohibited Changes

The following are constitutionally forbidden:

- Making pause optional
- Hiding or delaying exit
- Turning system health into user performance
- Automating continuation without explicit consent

---

## Interpretation Clause

When ambiguity exists,
interpretations that favour
slower cognition,
clearer structure,
and stronger exit rights
shall prevail.
